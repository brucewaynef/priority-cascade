# priority-cascade 🎓

![Tests](https://github.com/YOUR_USERNAME/priority-cascade/actions/workflows/tests.yml/badge.svg)
![Python](https://img.shields.io/badge/python-3.10%2B-blue)
![License](https://img.shields.io/badge/license-MIT-green)

Симулятор конкурсного зачисления в вуз по нескольким спискам приоритетов.

Если ты подавался на несколько специальностей одного вуза с разными
приоритетами — этот скрипт честно посчитает, кто из абитуриентов
"выше тебя" в конкурсном списке реально там останется, а кто уйдёт на
специальность с более высоким приоритетом (и освободит место тебе) —
то есть каскадом обрушит очередь вниз, отсюда и название.

Без этого скрипта люди делают это вручную: открывают десяток вкладок
с разными конкурсными списками и ищут каждого через Ctrl+F. Скрипт
делает то же самое для всех абитуриентов сразу — и без ошибок.

## Как это работает

Используется алгоритм **отложенного принятия** (deferred acceptance,
он же Gale–Shapley) со стороны абитуриентов:

1. Каждый абитуриент "предлагает себя" специальности с приоритетом 1.
2. Специальность оставляет только top-N (по количеству мест) абитуриентов
   по конкурсному баллу среди тех, кто её выбрал.
3. Отклонённые абитуриенты откатываются на следующий по списку приоритет.
4. Повторяем, пока распределение не станет стабильным.

Это именно то, что происходит в реальной системе вуза при перебросе мест
между волнами и приоритетами — просто без ручного Ctrl+F.

## Установка

```bash
git clone https://github.com/YOUR_USERNAME/priority-cascade.git
cd priority-cascade
pip install -r requirements.txt
```

`colorama` в requirements нужна только для цветного вывода в Windows-консоли,
без неё скрипт тоже работает (просто без цвета).

## Тесты

Логика проверена набором тестов на синтетических (выдуманных) данных —
никаких реальных персональных данных абитуриентов в репозитории нет.

```bash
pip install -r requirements-dev.txt
pytest tests/ -v
```

Тесты гоняются автоматически через GitHub Actions на каждый push (см.
`.github/workflows/tests.yml`).

## Формат входных данных

Один CSV-файл = один конкурсный список одной специальности (или одной
конкурсной группы — например, отдельно "общий конкурс" и "особая квота").

Разделитель `;`, кодировка `utf-8` (с BOM или без — не важно). Обязательные
колонки: `Порядковый номер`, `Приоритет конкурса`, `Код поступающего`.

**Имя файла** задаёт код специальности и число бюджетных мест:

```
<код_специальности>_<количество_мест>.csv
```

Примеры:
```
09.03.01_30.csv           -> специальность 09.03.01, 30 мест
09.03.01(1)_16.csv        -> та же специальность, другая конкурсная группа, 16 мест
prikladnaya_matematika_15.csv
```

Если у двух конкурсных групп совпадает код специальности — просто добавь
любой суффикс к коду в имени файла (например `(1)`), это не сломает логику:
ключом базы всегда служит имя файла целиком.

Положи все файлы в одну папку.

## Быстрый старт (на игрушечном примере)

В репозитории есть синтетические (полностью выдуманные) данные в
`examples/sample_data/` — можно попробовать сразу после клонирования,
без своих реальных баз:

```bash
python3 admission_checker.py examples/sample_data --my-code 1002 --all
```

В этом примере абитуриент `1001` формально стоит выше `1002` в списке
`spec_a` (1 место), но у него приоритет 1 в `spec_b` — и он туда реально
проходит по баллам. Место в `spec_a` освобождается, и `1002` проходит.
Ровно это скрипт и делает с твоими настоящими данными, только в масштабе
тысяч абитуриентов вместо пяти.

## Использование

```bash
# Базовый прогон — покажет твои приоритеты и итоговый вердикт
python3 admission_checker.py ./data --my-code 1234567

# + детальный список по конкретной специальности
python3 admission_checker.py ./data --my-code 1234567 --target 09.03.01_30

# + отчёт "кто ушёл выше тебя и куда"
python3 admission_checker.py ./data --my-code 1234567 --target 09.03.01_30 --mobility

# Детальный отчёт сразу по ВСЕМ специальностям
python3 admission_checker.py ./data --my-code 1234567 --all

# Экспорт итогового распределения
python3 admission_checker.py ./data --my-code 1234567 --export-json result.json --export-csv result.csv
```

`--my-code` — обязательный параметр. Скрипт нигде не хранит твой личный
код поступающего внутри файла — так безопаснее, если репозиторий публичный.

## Ограничения

Скрипт видит только те специальности, базы которых ему передали. Если у
кого-то из абитуриентов есть более приоритетный вариант в специальности,
данных по которой нет — скрипт не сможет это учесть. Чем больше баз
(специальностей) собрано, тем точнее результат.
