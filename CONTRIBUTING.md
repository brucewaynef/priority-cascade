# Contributing

PR и issues приветствуются. Пара правил:

1. **Никогда не коммить реальные конкурсные списки / CSV с персональными
   данными.** `.gitignore` уже блокирует `*.csv`, но перепроверяй перед
   пушем — это персональные данные реальных людей.
2. Перед PR прогони тесты: `pytest tests/ -v`. Новая логика — новый тест.
3. Код на Python 3.10+, тайпхинты приветствуются, но не обязательны для
   мелких правок.
4. Баги / идеи описывай в issues с примером (синтетическим!) CSV, который
   воспроизводит проблему.
